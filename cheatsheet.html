Docker
    • create — create a container from a selected image.
    • start — activate an existing container.
    • run — create a new container and immediately start it.
    • ls — list all existing containers.
    • inspect — provide detailed information about the selected container.
    • logs — display logs (event journal) in the console.
    • stop — attempt to stop the selected container by sending it a SIGTERM signal, asking it to terminate all activities and save user data. If it takes too long to respond, it then sends a SIGKILL signal to "kill" the process without saving data.
    • kill — performs the same task as the previous command, but skips the step of sending SIGTERM. Immediately shuts down the container, ignoring the saving of user data.
    • build — builds an image from scratch.
    • push — sends the image to a registry.
    • pull — fetches a ready-made image with the necessary parameters for operation.
    • ls — lists all existing images.
    • history — displays each layer of the image in retrospect, showing a series of useful information.
    • rm — deletes a Docker image from the system.
    • images — lists all Docker images found on the disk.
